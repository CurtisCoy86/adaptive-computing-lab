# Adaptive Computing Lab

Adaptive Computing Lab is a research and software development direction focused on building systems that adapt to the individual.

The goal is not to immediately build a full AI operating system, personal AGI, or fully autonomous life-management platform. The goal is to build small, testable software experiments that explore one central question:

> How can software observe human patterns, reduce cognitive load, adapt to individual needs, and become easier to live with over time?

Adaptive Computing Lab begins with practical experiments. Each experiment should be small enough to build, test, review, and either integrate or reject.

## Current Testbed

The first active testbed is **Shellwalker**.

Shellwalker began as a Python terminal survival RPG and has evolved into a neurodivergent life assistant. It currently includes:

- Medication tracking
- Task tracking
- Bill tracking
- Check-ins
- Check-in trends
- Startup daily brief
- Edit/remove commands
- Tests
- Git history

Shellwalker is not just an app. It is the first laboratory environment.

## Core Rule

Dream large. Build small. Test honestly. Integrate carefully.

## Research Cycle

Every experiment should follow this cycle:

1. Hypothesis
2. Input
3. Behavior
4. Test
5. Risk
6. Integration decision

## Agent Software Factory

Agents may be used to speed up research and development:

- Builder Agent
- Tester Agent
- Reviewer Agent
- Reporter Agent

Human review remains central. Agents do not decide what enters the project.

## First Research Direction

The first major direction is adaptive cognitive-load reduction inside Shellwalker.

Initial experiments include:

- Adaptive Daily Brief
- Overwhelm Detector
- One-Thing Mode
- Check-In Pattern Summary
- Agent-Built Feature Trial
